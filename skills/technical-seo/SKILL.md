# Technical SEO

A skill for auditing and improving the technical foundations that allow search engines and AI systems to crawl, render, index, understand, and trust a website.

This skill focuses on:

- crawlability
- indexability
- rendering
- site architecture
- Core Web Vitals
- structured data
- canonicalization
- internal linking
- sitemap quality
- technical SEO scalability
- AI-search readiness

## Core principles

### 1. Crawlability comes first

Search engines must be able to:

- discover URLs
- access pages
- render content
- follow links

Key areas:

- robots.txt
- internal linking
- crawl depth
- sitemap quality
- HTTP status codes

---

### 2. Indexability controls visibility

A page cannot rank if it cannot be indexed.

Always validate:

- canonical tags
- noindex directives
- duplicate content
- thin pages
- soft 404s
- redirect chains

---

### 3. Rendering matters

Modern websites often depend heavily on JavaScript.

Ensure important content is visible in:

- server-rendered HTML
- rendered HTML
- mobile rendering

Critical elements should not depend entirely on client-side rendering. :contentReference[oaicite:1]{index=1}

---

### 4. Site performance affects SEO

Core Web Vitals are important for:

- rankings
- user experience
- crawl efficiency
- AI-search usability

Focus on:

- LCP
- INP
- CLS

Target strong real-user performance. :contentReference[oaicite:2]{index=2}

---

### 5. Structured data improves understanding

Structured data helps search engines understand entities and page content. :contentReference[oaicite:3]{index=3}

Use structured data to improve:

- rich results
- entity understanding
- AI-search comprehension

Validate:
- syntax
- eligibility
- content alignment

---

### 6. AI-search systems require technical accessibility

AI-powered search systems rely heavily on:

- crawlability
- rendering
- structure
- semantic clarity
- content accessibility

Technical SEO supports both:
- traditional search
- AI retrieval systems

## Technical SEO audit workflow

### Step 1: Crawlability audit

Check:

- robots.txt
- blocked resources
- crawl depth
- orphan pages
- internal linking
- sitemap discoverability
- HTTP status codes
- redirect chains

Key questions:

- Can search engines discover important pages?
- Are crawl resources wasted on low-value URLs?
- Are critical assets blocked?

Use:
- Google Search Console
- crawl tools
- server logs if available

---

### Step 2: Indexability audit

Check:

- canonical tags
- duplicate content
- noindex directives
- index bloat
- thin content
- faceted URLs
- pagination handling
- parameter handling

Validate whether indexed URLs align with strategic pages.

---

### Step 3: Rendering audit

Check:

- JavaScript rendering
- rendered HTML output
- mobile rendering
- hydration issues
- lazy-loaded content
- hidden content
- rendering delays

Ensure important SEO elements appear in rendered HTML:

- titles
- canonicals
- meta robots
- structured data
- main content

---

### Step 4: Site architecture audit

Evaluate:

- URL structure
- crawl depth
- internal linking
- hub/spoke relationships
- navigation clarity
- content discoverability

Important pages should generally be reachable within a few clicks.

---

### Step 5: Core Web Vitals audit

Review:

- LCP
- INP
- CLS

Identify causes such as:

- render-blocking resources
- oversized JavaScript
- layout shifts
- slow server response
- unoptimized images
- excessive third-party scripts

Prioritize fixes with the highest user impact.

---

### Step 6: Structured data audit

Check:

- JSON-LD implementation
- schema eligibility
- syntax errors
- entity alignment
- visible-content alignment

Validate using:
- Rich Results Test
- Search Console enhancements reports

Do not add misleading or invisible schema. :contentReference[oaicite:4]{index=4}

---

### Step 7: Sitemap audit

Validate:

- sitemap freshness
- canonical URLs only
- HTTP 200 status pages only
- no redirected URLs
- no blocked URLs
- no orphan URLs

Large sites should use:
- segmented sitemaps
- sitemap indexes

---

### Step 8: AI-search readiness audit

Check whether pages are:

- easily retrievable
- clearly structured
- semantically understandable
- citation-friendly

Improve:
- heading hierarchy
- semantic structure
- answer clarity
- entity clarity
- machine readability

### Crawlability problems

Examples:

- blocked pages
- broken internal links
- excessive crawl depth
- orphan pages
- poor sitemap coverage

---

### Indexability problems

Examples:

- incorrect canonicals
- accidental noindex
- duplicate pages
- soft 404s
- parameter duplication

---

### Rendering problems

Examples:

- JS-only content
- delayed rendering
- missing rendered metadata
- hydration failures
- blocked JS assets

---

### Performance problems

Examples:

- poor Core Web Vitals
- oversized scripts
- image bloat
- slow server response
- layout instability

---

### Structured data problems

Examples:

- invalid schema
- mismatched content
- unsupported schema types
- missing required properties
